# 🚀 2단계 - 칸반 보드 관리(수정)

우아한테크코스 안드로이드 8기 레벨 1 미션, 칸반 보드 관리 미션을 수행하는 프로젝트입니다.

## 1단계 피드백 추가 반영

### 이벤트 처리 구조 개선

1단계에서 스낵바 이벤트를 `mutableStateOf`로 관리하던 구조의 한계를 인지하고 SharedFlow 기반으로 개선합니다. 상태 홀더는 이벤트 발생 사실만 방출하고, UI는 이를 구독하여 처리합니다.

## 2단계 기능 및 작업 명세

### Review 상태 추가 및 상태 전이 규칙 구현
- To Do, In Progress, Review, Done 네 가지 상태를 지원합니다.
- 상태 전이는 정해진 규칙에 따라서만 가능하며 규칙에 벗어난 전이를 시도하면 스낵바로 안내합니다.
  - To Do → In Progress
  - In Progress → To Do, Review
  - Review → In Progress, Done
  - Done → To Do

### 상태별 태스크 규칙 적용
- 태스크의 상태에 따라 삭제 가능 여부와 담당자 지정 필수 여부가 달라집니다.
  - To Do: 삭제 가능, 담당자 미지정 가능
  - In Progress: 삭제 가능, 담당자 지정 필수
  - Review: 삭제 불가, 담당자 지정 필수
  - Done: 삭제 불가, 담당자 지정 필수
- 담당자 없이 담당자가 필수인 상태로 이동하려 하면 스낵바로 안내합니다.
- 이 규칙들은 도메인이 직접 판단하도록 합니다.

### 태스크 수정/삭제 도메인 메서드 추가 및 담당자 선택 옵션화
- 칸반 보드에 태스크 수정·삭제 기능을 도메인 메서드로 추가합니다.
- 태스크 담당자를 선택하지 않을 수 있도록 옵셔널로 변경합니다.

### SharedFlow 기반 이벤트 처리 개선
- 스낵바 이벤트를 mutableStateOf 기반 상태 모델에서 SharedFlow 기반 이벤트 스트림으로 개선합니다.
- 수정/삭제/상태 전이 실패에 대한 이벤트를 추가합니다.

### 태스크 생성 화면과 수정 화면의 공통 컴포넌트 조합
- 생성과 수정은 동일한 입력 폼을 공유하고 모드에 따라 초기값과 버튼 구성만 달라집니다.

### 태스크 수정 및 삭제
- 카드를 클릭하면 수정 화면이 열리며 제목, 설명, 태그, 상태, 담당자를 변경할 수 있습니다.
- 수정 화면에서 상태 전이 규칙에 따라 이동 가능한 상태만 선택할 수 있습니다.
- 삭제 가능한 상태의 태스크만 삭제 버튼이 노출됩니다.
- 수정 및 삭제 결과는 스낵바로 안내합니다.

## 테스트 명세

### 단위 테스트
- **도메인 비즈니스 로직 검증**
  - 상태 전이 규칙이 의도대로 동작하는지 전체 경우의 수를 검증합니다.
  - 상태별 삭제 가능 여부, 담당자 필수 여부를 검증합니다.
  - 태스크 수정·삭제 시 불변성이 유지되는지 검증합니다.
- **상태 홀더 검증**
  - 태스크 수정·삭제 성공 및 실패 시 올바른 이벤트가 방출되는지 검증합니다.
- **폼 검증**
  - 선택된 상태에 따라 담당자 필수 여부 검증이 올바르게 동작하는지 검증합니다.
